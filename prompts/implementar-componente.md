# Implementar Componente

Leia os documentos-base do projeto e implemente apenas o componente solicitado.

Componente alvo: `<NOME_DO_COMPONENTE>`

Instrucoes:

1. Resuma a responsabilidade do componente.
2. Liste entradas, saidas, eventos e metricas.
3. Liste explicitamente as nao-responsabilidades.
4. Proponha a estrutura de diretorios e arquivos.
5. Implemente somente o minimo necessario para integrar o componente ao MVP.
6. Inclua logs estruturados minimos.
7. Inclua metricas minimas coerentes com `/docs/metricas.md`.
8. Inclua testes basicos.
9. Inclua README curto do componente.

Proibicoes:

* nao adicionar features fora do componente;
* nao criar abstracoes genericas sem uso imediato;
* nao acoplar diretamente com componentes nao necessarios;
* nao substituir contratos ja definidos em `/docs`.

Saida esperada:

* resumo do componente;
* plano de arquivos;
* implementacao;
* instrucoes de execucao;
* instrucoes de validacao;
* limitacoes.
